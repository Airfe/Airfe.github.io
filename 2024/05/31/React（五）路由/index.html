React（五）路由

react-router-dom

理解

  1. raact router以三个不同的包发布到npm上，分别为react-router核心库、react-router-dom库（包含了

    react-router，还额外添加了一些用于DOM的组件）、react-router-native（包含react-router，还额外添加

    一些用于ReactNative的api）

  2. 专门用来实现一个SPA应用，用于web端。

  3. 基于react的项目基本都会用到此库。

相关API

内置组件

  • <BrowserRouter> <HashRouter> <Route> <Redirect> <Link> <NavLink> <Switch>

其它

  • history对象、match对象、withRouter函数

基本使用

  1. 下载react-router-dom: npm install --save react-router-dom

  2. 找到导航区，把原来的a标签写成Link标签，如果需要设置选中的样式的话写成NavLink标签，实现点击地址栏变化但不跳转

    • Link标签实现

    • NavLink可以实现路由链接的高亮，通过activeClassName指定样式名

    • 为了避免重复性代码问题，一般可以自己封装一个一般组件MyNavLink. 就可以使用更简洁的方式实现导航区

    标签体内容是一个特殊的标签属性，通过this.props.children可以获取组件标签的标签体内容

  3. 找到展示区，写Route标签进行路径和路由组件的匹配

  4. 在最外层（也就是App组件的外层）包裹一个< BrowserRouter>或者< HashRouter>路由管理器，使规则生效

路由组件 VS 一般组件

  1. 写法不同

  2. 存放位置不同：

    • 一般组件：components
    • 路由组件：pages

  3. 接收到的props不同：

    • 一般组件：写组件标签时传递了什么，就能收到什么
    • 路由组件：接收到三个固定的属性

比如，浏览器会向 localhost:3000/att 下请求 css 样式资源，这并不是我们想要的，因为我们的样式存放于public（local:3000/）的 CSS 文件夹中。

我们有几种方法，可以解决这个问题

  1. 将样式引入的路径改成绝对路径

  2. 引入样式文件时不带 .

push与replace模式

默认情况Link开的都是push模式，每次点击跳转，都会向栈中压入一个新的地址，在点击返回时，可以返回到上一个打开的地址。

开启replace，将当前地址替换成点击的地址，也就是替换了新的栈顶

BrowserRouter 和 HashRouter 的区别

  1. 底层实现原理
    • BrowserRouter 来说它使用的是 React 为它封装的 history API,操作这些 API 来实现路由的保存等操作,兼容性差
    • 通过监听hah值的变化来更新页面内容
  2. 地址栏的表现形式
    • HashRouter 的路径中包含 #
  3. 刷新后路由 state 参数改变
    • 在BrowserRouter 中，使用state传参方式是 保存在history 对象中，刷新不会丢失
    • 但是使用HashRouter， 刷新会丢失 state参数

router 6相关

目前最新版本react router的安装就是6

6中移除了一些组件，新增了一些内置组件和路由相关的hooks

  1. 移除了<Switch>,引入了替代者：<Routes>

  2. 引入了<Navigate> ，只要 <Navigate> 组件被渲染，就会修改路径，切换视图。可用于路由重定向

  1. 引入useRoutes() 路由表，接受一个数组，即路由规则。
  2. 引入<Outlet> ，设置子路由的路由出口，即在何处渲染子路由。
  1. 改变自定义<NavLink> 路由高亮样式的方式，需要通过函数返回
  1. 路由传参

params、search的路由注册和路由传参都与router5一样，只是router6里面推荐使用函数组件，推广hooks，因此取出参数的方式有所变化。

  • params 取参：使用useParams()

  • search取参：使用 useSearchParams()

state的路由注册与router5一样，路由传参做了简化，取参也用了hook方式

  • state路由传参

  • state取参：使用 useLocation()

  1. 编程式路由导航

router5的编程式路由导航依赖于this.props.history，还是因为router6倡导函数组件，所以提出了

useNavigate() 函数实现编程式路由导航。

有两种参数传递方式。

  • 第一种方式传递两个参数：路由和相关参数。参数只能设置 replace 和 state。想要传递 params 和 search 参数直接在路由带上。
  • 第二种方式传递数字，代表前进或后退几步。

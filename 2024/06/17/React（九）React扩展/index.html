组件优化

Component的2个问题

  1. 只要执行setState()即使不改变状态数据（传空）, 组件也会重新render() ==> 效率低
  2. 只当前组件重新render(), 就会自动重新render子组件，纵使子组件没有用到父组件的任何数据 ==> 效率低

效率高的做法

只有当组件的state或props数据发生改变时才重新render()

原因

Component中的shouldComponentUpdate()总是返回true

解决

  1. 法1: 重写shouldComponentUpdate()方法，比较新旧state或props数据, 如果有变化才返回true, 如果没有返回false

  2. 法2：使用PureComponent，PureComponent重写了shouldComponentUpdate(), 只有state或props数据有变化才返回true

    注意: 只是进行state和props数据的浅比较, 如果只是数据对象内部数据变了, 返回false 。不要直接修改state数据, 而是要产生新数据

render props

类似于vue中的插槽

向组件内部动态传入带内容的结构(标签)有两种方法

1. children props

通过过组件标签体传入结构，组件标签内的内容会在子组件的props.children上

2. render props

通过组件标签属性render传入结构，而且可以携带数据，一般用render函数属性，但是写成其他的也可以。

结构在props.render上

错误边界

理解

错误边界(Error boundary)：用来捕获后代组件错误，渲染出备用页面

特点：只能捕获后代组件生命周期产生的错误，不能捕获自己组件产生的错误和其他组件在合成事件、定时器中产生的错误

使用方式

getDerivedStateFromError配合componentDidCatch

React（六）Hooks

Hooks

1. React Hook/Hooks是什么?

Hook是React 16.8.0版本增加的新特性/新语法，可以让我们在函数组件中使用 state 以及其他的 React 特性

常用的Hook

  1. State Hook: React.useState()
  2. Effect Hook: React.useEffect()
  3. Ref Hook: React.useRef()
  4. Context Hook：React.useContext()
  5. Reducer Hook：React.useReducer()

和性能优化相关的hook

  1. Memo Hook：React.useMemo()
  2. Callback Hook：React.useCallback()
  3. Transition Hook：**React.useTransition() **
  4. DeferredValue Hook：React.useDeferredValue()

2. State Hook

State Hook让函数组件也可以有state状态, 并进行状态数据的读写操作。

使用

const [xxx, setXxx] = React.useState(initValue)

  • 参数: 第一次初始化指定的值在内部作缓存
  • 返回值: 包含2个元素的数组, 第1个为内部当前状态值, 第2个为更新状态值的函数

2种写法

  • setXxx(newValue): 参数为非函数值, 直接指定新的状态值, 内部用其覆盖原来的状态值

  • setXxx(value => newValue): 参数为函数, 接收原本的状态值, 返回新的状态值, 内部用其覆盖原来的状态值

3. Effect Hook

Effect Hook 可以让你在函数组件中执行副作用操作(用于模拟类组件中的生命周期钩子)。

使用

React中的一些副作用操作

  • 发ajax请求数据获取
  • 设置订阅 / 启动定时器
  • 手动更改真实DOM

参数分析

useEffect Hook 的第二个数组参数有监听的意味。首先useEffect中的内容都会在render（）后执行一次，后面什么时候再执行就看第二个参数。

  • 传[]，不监听任何变量，只会在第一次render()后执行，相当于 componentDidMount()
  • 如果不传则监听所有变量，传某一个变量就是只检测这个变量。类似于componentDidUpdate()
  • 第一个参数（函数）返回的函数执行的时机除了每一次useEffect副作用函数执行时执行，还会在组件将卸载时执行，相当于componentWillUnmount()

useEffect vs useLayoutEffect

  • 回调函数的执行时机不同：useEffect是浏览器重新绘制屏幕之后触发；useLayoutEffect 中的回调函数在浏览器重新绘制屏幕之前触发。
  • 执行过程不同：useEffect 中的回调函数异步执行，不阻塞浏览器绘制；useLayoutEffect 中的回调函数同步执行s，阻塞浏览器重新绘制 。

4. Ref Hook

Ref Hook可以在函数组件中存储/查找组件内的标签或任意其它数据，功能与React.createRef()一样。

使用

  1. 声明一个标签容器 const refContainer = useRef()
  2. 将标签容器赋给目标节点<input type="text" ref={refContainer}/>
  3. 获取目标节点的数据alert(refContainer.current.value)

与React.createRef()一样，如果目标节点是一个函数组件，函数式组件是没有实例的，直接用ref接不到东西。

解决方法：在目标组件中使用React.forwardRef() 包裹，使其可以接收ref属性，再用useImperativeHandle暴露出对外的内容。

6. Reducer Hook

当状态更新逻辑较复杂时可以考虑使用 useReducer。 useReducer 可以更好的描述“如何更新状态” ，让代码逻辑

更清晰，代码行为更易预测。

参数分析

  • reducer 是一个函数，类似于 (prevState, action) => newState。形参 prevState 表示旧状态，形参 action 表示本次的行为，返回值 newState 表示处理完毕后的新状态。

  • initState 表示初始状态，也就是默认值。

  • initAction 是进行状态初始化时候的处理函数，它是可选的，如果提供了 initAction 函数，则会把 initState 传递给 initAction 函数进行处理，initAction 的返回值会被当做初始状态。

  • 返回值 state 是状态值。

  • dispatch 是更新 state 的方法，传入 的参数作为action ，reducer 接受到dispatch传过来的action进行更新操作。

使用

7. Memo Hook

使用React.memo可以防止子组件不必要的重新渲染，使用useMemo可以避免自身函数不必要的重新执行。

参数分析

useMemo 的第二个数组参数有监听的意味

  • 如果不传则监听所有变量
  • 传某一个变量就是只检测这个变量。类似于componentDidUpdate()
  • 传[]，不监听任何变量，只会在第一次render()后执行，相当于 componentDidMount()
  • 返回值是一个计算好的值，这个值有缓存，只有依赖项变化才重新计算

使用

8. Callback Hook

useMemo 能够达到缓存某个变量值的效果，而useCallback 用来对组件内的函数进行缓存，防止组件每次 render 时反复创建相同的函数，能够节省内存开销，提高性能。

参数分析

  • cb 是一个函数，用于处理业务逻辑，这个 cb 就是需要被缓存的函数

  • array 是依赖项列表，当 array 中的依赖项变化时才会重新执行 useCallback。

    • a. 如果省略 array，则每次更新都会重新计算
    • b. 如果 array 为空数组，则只会在组件第一次初始化的时候计算一次
    • c. 如果 array 不为空数组，则只有当依赖项的值变化时，才会重新计算

使用

9. Transition Hook

useTransition 可以将一个更新转为低优先级更新，使其可以被打断，不阻塞 UI 对用户操作的响应，能够提高用户的使用体验。它常用于优化视图切换时的用户体验 。

正常情况下页面渲染完，用户才可以去进行ui操作

参数分析

返回值（数组）：

  • isPending 布尔值：是否存在待处理的 transition，如果值为 true，说明页面上存在待渲染的部分，可以给用户展示一个加载的提示；

  • startTransition 函数：调用此函数，可以把状态的更新标记为低优先级的，不阻塞 UI 对用户操作的响应；即startTransition里面的setXxx操作会变成低优先级。

使用

注意

  1. 传递给 startTransition 的函数必须是同步的。React 会立即执行此函数，并将在其执行期间发生的所有状态更新标记为 transition。如果在其执行期间，尝试稍后执行状态更新（例如在一个定时器中执行状态更新），这些状态更新不会被标记为 transition；
  2. 标记为 transition 的状态更新将被其他状态更新打断。例如在 transition 中更新图表组件，并在图表组件仍在重新渲染时继续在输入框中输入，React 将首先处理输入框的更新，之后再重新启动对图表组件的渲染工作；
  3. transition 更新不能用于控制文本输入，会造成中间状态的丢失。

10. DeferredValue Hook

useDeferredValue 提供一个 state 的延迟版本，根据其返回的延迟的 state 能够推迟更新 UI 中的某一部分，从而达到性能优化的目的。

参数分析

useDeferredValue 的返回值为一个延迟版的状态：

  • 在组件首次渲染期间，返回值将与传入的值相同

  • 在组件更新期间，React 将首先使用旧值重新渲染 UI 结构，这能够跳过某些复杂组件的 rerender，从而提高渲染效率。随后，React 将使用新值更新 deferredValue，并在后台使用新值重新渲染是一个低优先级的更新。这也意味着，如果在后台使用新值更新时 value 再次改变，它将打断那次更新。

使用

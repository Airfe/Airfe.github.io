异步编程（一）Promise

同步和异步

同步：对应内存中顺序执行的处理器指令。

异步：类似于系统中断，即当前进程 外部的实体可以触发代码执行。（异步代码内部还没执行，下面的同步代码已经执行了）

异步操作经常是必要的，因为强制进程等待一个长时间的操作通常是不可行的。

以往的异步编程

异步代码的返回值，只能在异步代码执行结束之后调用回调函数传递。

异步代码的异常处理

异步代码的嵌套（包括各级的异常处理）

嵌套层级的不断增加===>回调地域

Promise

什么是promise

是现在Javascript的一种常用优秀的异步解决方案，以前我们需要拿到异步代码的返回值，只能在异步代码里面调回调函数，如果回调函数里面又有异步代码就又有回调函数，不断的嵌套，就会形成俗称的回调地域。promise就可以很好地解决这个问题。

其他常用的异步方案还有async/await。

期约的主要两大用途：知道异步代码的执行情况（状态）、实际生成某个值。

promise状态

每一个 Promise的实例都有一个状态，一共有三种状态。

  • resolved（解决）
  • rejected（拒绝）
  • pending（待定）

在待定状态下，期约可以落定为代表成功或者失败的状态，无论落定为哪种状态都是不可逆的。

在执行器函数内部调用内置的resolve()和reject()函数可以改变Promise的状态，同时也可以传递参数，这个参数就叫做解决值/拒绝值。

外部的同步代码无法读取或修改期约的状态，异步代码只能通过期约的方法（异步结构）与之交互。

执行器

执行器的作用如下

  • 初始化Promise的行为

  • 控制Promise的状态转换（通过调用两个函数参数实现）

  • 注意

    执行器函数是其实是同步的，也就是说执行器函数的代码是会比new…后面的代码先执行的

    但是执行器函数里面允许异步代码，比如resolve()、reject()、setTimeout()

    直接抛出异常可以理解为执行失败=拒绝=reject()，故执行器里面直接抛出错误外面的同步代码是捕获不到的，只能通过异步结构拿到。

    可以理解为异步代码是由一个函数包裹住的，里面的参数外面的代码都拿不到，只有函数内（异步代码）才可以取得。

解决值和拒绝值

每个期约只要状态切换为兑现，就会有一个私有的内部值（value）。状态切换为拒绝，就会有一个私有的内部理由（reason）。无论是值还是理由，都是包含原始值或对象的不可修改的引用。二者都是可选的，而且默认值为 undefined。

比如，假设期约向服务器发送一个 HTTP 请求并预定会返回一个 JSON。如果请求返回范围在 200299 的状态码，则足以让期约的状态变为兑现。此时期约内部就可以收到一个 JSON 字符串。类似地，如果请求返回的状态码不在 200299 这个范围内，那么就会把期约状态切换为拒绝。此时拒绝的理由可能是一个 Error对象，包含着 HTTP 状态码及相关错误消息。

到了落定状态后，期约会提供其解决值（如果兑现）或其拒绝理由（如果拒绝）给相关状态的处理程序。

静态方法

Promise.resolve()

创建一个已经解决的Promise

入参是任意类型，返回值是一个Promise

转换规则：

  • 入参是基础数据类型

    返回值：一个解决值为这个基础值的已解决的Promise

  • 入参就是一个Promise

    返回值：直接返回入参的Promise

  • 入参是一个异常

    返回值：一个解决值为这个异常的已解决的Promise

  • 多个参数

    返回值：把第一个参数按照上述规则包装返回

Promise.reject()

入参是任意类型，返回值是一个Promise

  • 入参是基础数据类型

    返回值：一个拒绝值为这个基础值的已拒绝的Promise

  • 入参就是一个Promise

    返回值：一个拒绝值为这个入参的已拒绝的Promise

  • 入参是一个异常

    返回值：一个拒绝值为这个入参的已拒绝的Promise

  • 多个参数

    返回值：把第一个参数按照上述规则包装返回

Promise.race()、Promise.all()

Promise 类提供两个将多个期约实例组合成一个期约的静态方法：Promise.all()和 Promise.race()。而合成后期约的行为取决于内部期约的行为。

  • 入参：包含promise的可迭代对象（一般就是promise数组）

  • 返回值：一个promise

  • 状态改变：

    • promise.race([xxx,xxx])什么时候改变状态，取决于入参哪个先改变状态。
    • promise.all([xxx,xxx])需要入参的所有promise都resolved解决之后才改变，有一个不是的话状态直接转变为rejected、pending

  • 应用：

    接口超时拒绝

实例方法

期约实例的方法是连接外部同步代码与内部异步代码之间的桥梁。这些方法可以

  • 访问异步操作返回的数据
  • 处理期约成功和失败的结果
  • 连续对期约求值
  • 添加只有期约进入终止状态时才会执行的代码

.then()

接收参数：两个，分别是onResolved和onRejected，分别代表这个Promise状态被解决的时候执行和被拒绝的时候被执行

返回值：一个Promise

入参规则：两个参数都可选。

  • 超过两个参数，自动忽略多余参数
  • 入参类型不是函数也会被自动忽略
  • 可以直传递第一个onResolved参数，不填第二个参数，处理成功的情况
  • 但是如果想只传递一个onRejected处理程序，第一个不能不写，需要写个占位的空函数或其他无意义的非函数入参。

因为Promise只能转换为最终状态一次，故这两个参数对应的函数操作一定是互斥的。只会执行其中一个

返回规则：

  • 入参两个都没填，就直接返回上一个promise
  • 只传入了解决处理函数，返回一个已解决且解决值为处理函数的返回值的promise，如果处理函数返回值是一个promise，也直接返回这个promise
  • 只传入了拒绝处理函数，返回一个已解决且解决值为处理函数的返回值的promise，如果处理函数返回值是一个promise，也直接返回这个promise

没有返回值（就是undefined）和返回显式的基础数据类型一样（包括异常），都是直接包装成一个已解决的并且解决值为这些基础数据类型的promise

onRejected的返回值包装成Promise的规则和onResolved包装规则是一样的

.catch

是onRejected的另一种写法

.finally

在promise转换为已拒绝或者已解决时都会执行，一般用于兜底、添加清理代码

非重入promise方法

是一个概念，当promise状态改变时对应的处理程序，比如onResolve、onRejected、.catch等不是立马被执行的，而是放到一个异步时事件队列中（先进先出），一般要这个队列的事件是在同步代码之后执行。

git命令

GIT命令

  • **git branch ** 检查自己所处分支
  • git checkout -b user 新建一个user分支并切换到user分支
  • **git status ** 检查所处分支文件状态，绿色代表在暂存区
  • git add . 将修改和新增的文件统一放在暂存区
  • git reset +文件名 将暂存区的文件取消暂存或切换到指定版本
  • git commit -m “完成用户列表功能的开发” -m添加一个提交消息， 将当前分支提交到仓库中，运行完这句话以后查看状态git status ，当前工作目录应该是干净的
  • git push -u origin user 把本地的分支推送到云端仓库origin中，取名也为user
  • git checkout master 切换到master目录
  • git merge user 将user分支合并到主分支
  • **git push ** 将本地的master推送到云端
  • git pull 拉取远程代码
  • git clone 克隆项目
  • git log 查看日志，会显示commit记录，和版本标识可以用git reset –hard 版本标识 回到相应的版本

文件状态

Git工作区中的文件存在两种状态:

  • untracked 未跟踪(未被纳入版本控制)
  • tracked已跟踪( 被纳入版本控制)
    • Unmodified未修改状态
    • Modified 已修改状态
    • Staged已暂存状态

标签操作

  • **git tag ** 列出已有的标签
  • git tag 标签名 创建标签
  • git push origin 标签名 将标签推送至远程仓库
  • git checkout -b 新建分支名 标签名 检出标签（新建一个创库保存该标签状态）

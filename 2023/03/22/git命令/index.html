git命令

GIT命令

enter 逐行浏览，q 退出分页器返回命令行

1.clone

  • 克隆项目：git clone

2.branch

  • 查看当前所处分支： git branch

  • 显示所有远程分支： git branch -r

  • 新建一个分支：git branch newbranchname，基于当前分支新建

3.checkout

  • 切换分支：git checkout branchname / git switch branchname
  • 基于mian分支新建并切换到分支：git checkout -b dev_airfehuang main

4.status

  • 查看当前工作区是否干净，显示暂存区状态：git status

    • “Changes to be committed” 部分显示已添加到暂存区的文件（使用 git add 命令）。

    • “Changes not staged for commit” 部分显示已修改但未添加到暂存区的文件。

    • “Untracked files” 部分显示未被 Git 追踪的文件

5.add

  • 提交修改后的代码到暂存区：git add file
    • 如果要提交所有修改后的代码 git add .

6.commit

  • 提交暂存区代码到本地仓库：git commit -m “messgae”

7.log

  • 查看某个分支的提交记录：git log 分支名称，会显示commit的哈希值
    • git log -n 指定输出的提交记录条数：

8.push

  • 推送到远端分支：git push
  • 将本地的分支版本上传到远程合并：git push -u origin dev_airfehuang

9.fetch

  • 获取远程仓库的所有分支和提交（但不合并），确保本地的远程分支引用是最新的，避免合并（merge）后不一致： git fetch

10.merge

  • 合并远程分支到当前目录：git merge 远程分支

    git pull = git fetch + git merge

11.pull

  • 默认拉取关联的远程分支合并到本地当前分支：git pull

  • 指定拉取远程分支dev_kevin合到本地当前分支：git pull origin dev_kevin

12.revert

  • git revert hash ,创新一个新提交来做hash提交的更改逆操作。比如hash里面是新增一行代码，那么revert就会删除这行代码。

    git revert 不会修改提交历史，而是创建一个新的提交。

13.reset

影响本地git记录的HEAD指针，和

  • git reset –soft hash , 本地HEAD指针挪到指定hash，暂存区不更改，文件不更改。重新提交，解决冲突后合并。
  • git reset –mixed hash ,本地HEAD指针挪到指定hash，只是暂存区清掉了，文件不更改。重新add、提交，解决冲突后合并。
  • git reset –hard hash , 本地HEAD指针挪到指定hash，暂存区清掉，文件更改。重新写代码、add、提交，解决冲突后合并。
  • git reset –keep hash , 会尝试mixed，尝试失败则命令执行失败。

所以如果想撤销已经提交的commit，也可以通过reset先修改本地HEAD指针重新写代码提交，当然需要与远端的进行合并。

git reset 可以修改提交历史，暂存区和工作目录

文件状态

Git工作区中的文件存在两种状态:

  • untracked 未跟踪(未被纳入版本控制)
  • tracked已跟踪( 被纳入版本控制)
    • Unmodified未修改状态
    • Modified 已修改状态
    • Staged已暂存状态

标签操作

  • git tag 列出已有的标签
  • git tag 标签名 创建标签
  • git push origin 标签名 将标签推送至远程仓库
  • git checkout -b 新建分支名 标签名 检出标签（新建一个创库保存该标签状态）

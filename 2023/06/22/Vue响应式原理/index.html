Vue响应式原理

所谓 vue 的响应式，即组件 data 的数据一旦变化，就会立刻触发视图的更新。

Vue检测数据的原理

Vue监测对象数据改变的原理

  1. vm会将data加工后赋值给自身的属性_data，且始终保持一致，并且直接通过defineproperty()加上_data中的属性
  2. 当读取vm.name，其实是调用了vm上的getter，把vm._data里面的name返回
  3. 当修改vm.name，其实是调用了vm上的setter，把vm._data里面的name修改

1中加工后，_data中的数据不再直接给，而是通过get、set方法，当读取或者修改_data中数据时就会调用get、set方法，set方法中写了一个重新解析模板的调用，会引起页面变化。故vue能做到响应式。

vue更完善的是，当data中嵌套有对象时，vue会一直递归的创建getter、setter，直到非对象。

vm将源数据data中的数据加工（getter、setter）后存储到下划线data上，进行实际的读取和赋值响应式更新。同时又把数据返回到vm上方便模板里面直接用，而不是繁琐的。

data中是源数据并且vm.data不存在，下划线data是有数据劫持的，并且有vm._data。

Vue.set的使用

事先没有在data中写的对象属性，如果以后想动态添加：

直接用vm._data.student_sex是不管用的，这种方法不会创建getter、setter方法（不是系统用defineProperty创建的）

更不可能直接在vm上添加，vm.sex，因为vm上的属性也是动态的从vm._data上取的，自己写的vm.sex一方面不会有getter、setter方法去拿，另一方面就算能去vm._data上拿，vm._data上也没有对应getter、setter方法。

Vue提供了相关的api实现动态向对象里添加属性(具有响应特性，数据变化能反映到页面)、修改数组的方法：

Vue.set(target, key , val)

另外一个在vm上可以有相同作用的api是vm.$set，

注意对象不能是Vue实例或者Vue实例的根数据对象。即target不能是vm或vm._data

Vue监测数组数据改变的原理

vue不会给数组里面的每一项创建getter、 setter，也就是说通过索引值修改数组里的某一项，不是响应式的。只有通过vue提供的包装过的数组方法，才是响应式的关于vue数组方法详见集合引用类型Array

但是对data中的数组操作这些方法调用的并不是原始Array.prototype上的方法，是经过vue封装了的，vue做了两件事，第一件事就是实现跟Array.prototype上方法同样的功能（改变数组），第二间事情是重新解析模板生成虚拟DOM等等。

重新解析模板

每个组件实例都对应一个 watcher 实例，它会在组件渲染的过程中把属性记录为依赖。之后当依赖项的 setter 触发时，会通知 watcher重新计算，从而使它关联的组件重新渲染。

Vue2的基本原理（总结）

借助于defineproperty实现数据代理和数据劫持，把data里的属性都设置了getter、setter, 在getter中除了简单的把值返回，Vue实例的观察程序实例watcher会进行收集依赖，也就是标签和数据属性的对应关系。当这些依赖的setter被调用时，watcher会重新计算，进行视图更新。

缺陷

Vue2使用Object.defineProperty进行数据拦截，但有些时候无法拦截到。有以下缺陷

1）深度监听，需要递归到底，一次性计算量大

在遍历对象或数组时，需要进行深度监听，即需要递归到底，这会使得一次性计算量非常大。（这个问题在 vue3.0 中已经解决，vue3.0使用proxy，其解决原理是不一定要一次性递归，而是可以我们什么时候用，什么时候再递归。这个将放在后面的文章中讲解）

2）无法监听对象新增属性/删除属性

Object.defineProperty() 在进行新增对象的属性和删除属性时，视图是无法进行更新的，也就是数据监听不到，这一点在平常的开发中需要特别注意！否则有时候我们在取数据时总会莫名其妙地都不知道自己错在哪里。通常解决这个问题的方法是，使用 Vue.set 和 Vue.delete 来进行新增属性和删除属性，这样就可以解决数据无法监听的问题。

3）无法原生监听数组，需要特殊处理

Object.defineProperty() 这个 API 本身无法监听原生数组，下标方式修改数组数不会触发组件的重新渲染，需要通过重新定义数组原型的方式，来对数组进行数据监听。

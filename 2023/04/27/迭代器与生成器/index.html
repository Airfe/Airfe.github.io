迭代器与生成器

迭代器

定义

迭代：对一组固定的数据结构，按照一定顺序，多次执行某段代码块。

在ES6之前，执行迭代操作需要借助循环或者其他辅助结构才能做到。不同的数据结构执行迭代需要开发者写不同的循环和控制逻辑。

比如对象需要使用Object.keys()然后对key数组进行遍历循环；需要手动控制迭代的结束（数组通过的.length长度进行控制什么时候结束）。

不同的数据结构的迭代是该数据结构的固有逻辑，这类特定的数据结构（实现了可迭代协议的数据结构）都统一实现了一套逻辑或规范。

这套规范可以让开发者只关心迭代逻辑，不需要去控制循环和结束。迭代器调用*.next()*时需要帮我们做的事情有循环逻辑、终止逻辑、固有属性。

由于可能存在一个变量，执行多次迭代逻辑，并且每次迭代互不影响。所以每个支持可迭代协议的数据类型都保存了一个迭代器构造函数数，每次执行迭代时生成

一个新的迭代器。每个可迭代数据类型都有一个[Symbol.iterator]属性，通过调用属性引用的迭代器构造函数生成一个新的迭代器。

这个构造函数为了避免冲突和重名，使用了一个避免重名的数据类型Symbol类型。

可迭代协议

JavaScript中实现了这套规范(协议)的有以下数据类型：

  • 字符串
  • 数组
  • 映射
  • arguments对象
  • NodeList等DOM集合类型

没有对象！

支持迭代器模式的原生语言特性：

  • for-of循环
  • 数组解构
  • 扩展操作符
  • Array.from()
  • 创建集合
  • 创建映射
  • Promise.all()接收由期约组成的可迭代对象
  • Promise.race()接收由期约组成的可迭代对象
  • yield* 操作符，在生成器中使用

自定义可迭代对象

给对象加上简单的自定义迭代器

有两种情况均会调用迭代器的return函数，return函数返回值也是个有标识迭代结束done变量为true的对象

return函数相当干是一个声明周期钩子，在提前退出时会调用

  • for-of中的终止语句（break、continue、return、throw）
  • .next方法中的提前退出逻辑

生成器

定义

函数名称前面带一个星号表示它是一个生成器。（除了箭头函数）

生成器函数的返回值是一个生成器对象，该对象实现了迭代器协议，是一个可迭代对象。故该对象拥有可迭代对象的特征：

  • 拥有一个.next()函数
  • 返回值是一个包含标识下一个迭代值的value和表示是否结束done的对象
  • 还有一个可选的一个提前退出的return函数

迭代器对象是在数据结构的迭代器构造函数中声明next()方法逻辑；

生成器对象中的.next()函数返回值的返回逻辑在生成器函数中定义，生成器函数返回什么就是.next()的value。

如果生成器函数中没有返回值没有return语句，生成器对象在调用一次.next()函数就会返回{done:true,value:undefined}的值

如果有return语句，.next()返回的value属性 就是函数的返回值done为true

如果要对生成器对象调用多次.next()方法，生成多个迭代值怎么办? return 语句后面的语句就不执行了啊?

一个新的语法关键字yield由此而来

yield

yield关键字理解起来就是只可以在生成器函数中使用的高配版的return语句，每次执行到yield语句就会执行停止，返回yield后面的值，同时这个生成器函数的执

行上下文会被保留，在下次调用.next()函数时会执行之后的代码。

  • 暂停函数执行，返回返回值

常见使用场景1

常见使用场景2 利用生成器函数返回一个可迭代对象

  • 接收参数

除了暂停函数执行，返回值。yield关键字还可以作为函数的中间参数使用，下一个.next()函数的第一个调用入参，会传给上一次让函数暂停的yield关键字*。

  • 给yield语法增加星号，可以将其后面的可迭代对象序列化为一连串值

遇到yield语句，可以将yield理解为一个for-of循环，yield = for-of*

  • 当做默认迭代器

对象的.next()方法返回值为yield的返回值，yield又解析了可迭代数据类型数组this.values

终止生成器对象

  • return
  • throw

和迭代器不一样的是，生成器调用这两个函数可以强制生成器对象进入中止状态，done会变成true，value就是return语句传入的参数。

而throw语句则表示传入生成器一个异常，如果生成器内部没有接住这个异常，生成器对象会强制进入中止状态。

但是如果内部有catch语法并且接住了这个异常，则生成器则不会关闭。
